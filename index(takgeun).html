Day 3 학습내용

  1. 기본 개념
    • 인터넷 : 웹이 나오기 전부터 만들어진 컴퓨터 연결 네트워크
    • 웹 : 인터넷을 활용하는 응용서비스들 중의 하나, 서버와 브라우저 둘이 정보를 전달하고 공유하는 방식으로 운영.
    • 웹 발달단계 :
      • 1.0 : 문서를 표현하는 정적(static)인 웹사이트(89~90년대 중반까지)
      • 2.0 : 동적인 페이지가 나오기 시작 - 자바스크립트의 출현 때문
      • 3.0 : 더 화려하게 움직일 수 있도록 만드는 애니메이션 효과...CGI(Common Gateway Interface)
      • 4.0 : HTML5이 표준, 자바스크립트도 표준, 문서를 공유하여 모든 데이터 종류를 주고 받을 수 있는 시대 도래.

  3. Server

    • 서버
      • 서버는 일반적으로 클라이언트라고 불리는 사용자에게 서비스를 제공하는 소프트웨어 또는 하드웨어입니다.

      하드웨어 서버는 네트워크의 공유 컴퓨터로서, 대개 강력하고 데이터 센터에 저장됩니다. 소프트웨어 서버(하드웨어 서버에서 자주 실행됨)는 클라이언트 프로그램에 서비스를 제공하는 프로그램 또는 사용자 클라이언트의 user interface를 말합니다.


    • 웹클라이언트
    • 사용자와 컴퓨터 간에 대화할 수 있도록 모양새를 갖추는 것
    • 웹서버에 문서, 정보 등을 요청하고 받아서 사용자에게 출력해주는 역할을 하는 것


  4. 도메인과 URL / URI

    • IP(Internet Protocol) 주소 : .(점) 3개 사이를 숫자로 표현해서 번호를 매긴 것
    • 도메인 주소 : IP주소를 알아보기 쉽게 하기 위해 의미를 가진 뚜껑을 씌운 것
      • URI(Uniform Resource Identifier) : 리소스를 가리키는 모든 종류의 문자열
      • URL(Uniform Resource Locator) : 인터넷에서 웹페이지, 이미지, 비디오 등의 '위치'를 가리키는 문자열
        • 프로토콜              http://
        • 서버주소              127.0.0.1
        • TCP/IP 포트번호        :5500
        • 경로명                /01_html/
        • 파일이름              1.html


  5. 도메인주소를 입력했을 때 클라이언트(브라우저)는 무슨 일을 하는가
    • 브라우저의 구조

      • - 출처 : https://beomy.github.io/tech/browser/browser-rendering/

    • 렌더링 엔진이 하는 일

      1. 렌더 트리 생성 (DOM + CSSOM)
        • 가.DOM TREE
        • 나.CSSOM TREE
        • 다.렌더 트리 생성 (DOM + CSSOM)
      2. 레이아웃 - 렌더 트리가 생성되고, 기기의 뷰포트 내에서 렌더 트리의 노드가 정확한 위치와 크기를 계산하는 과정
      3. 페인트 - 렌더 트리의 각 노드를 화면의 실제 픽셀로 나타내는 과정
      4. 리플로우(Reflow), 리페인트(Repaint)
        사용자가 웹 페이지에 처음 접속을 하면, 렌더링 과정을 거쳐서 화면에 모든 요소가 그려집니다. 이후에 사용자의 액션에서 발생되는 이벤트로 인해서 새로운 HTML 요소가 추가되거나, 기존 요소의 스타일이 바뀌거나 하는 변경이 일어나게 됩니다. 이런 변경을 통해 영향을 받게되는 모든 노드에 대해서 렌더링 트리 생성과 레이아웃 과정을 다시 수행하게 됩니다.

▼ GET과 POST
https://section.blog.naver.com/BlogHome.naver?directoryNo=0¤tPage=1&groupId=0

GET : 클라이언트가 서버에 요청을 할 때 URL 주소 뒤에 붙여서 요청하는 방식입니다.
URL 끝에 ?를 붙이고 그 뒤에 변수명 = 값& 변수명 = 값 이어서 붙여나갑니다.


GET일 때 할 수 있는 것
  • 히스토리에 그대로 남아요
  • 캐시가 가능하다
  • 북마크 된다
  • 길이에 제한이 있다(길면 BODY로 받는 것이 좋다)

POST : 클라이언트가 서버에 요청을 할 때 BODY에 숨겨서 보내는 방식입니다.
  • 긴 내용을 보낼 때 사용한다
  • body의 타입은 Content-Type 헤더에 따라서 결정이 됩니다
  • 프라이버시 보호가 된다
  • 브라우저 히스토리에 남지 않는다
  • 데이터 길이에 제한이 없다
